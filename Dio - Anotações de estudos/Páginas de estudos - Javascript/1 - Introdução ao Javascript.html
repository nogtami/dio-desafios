Estudo de JavaScript

Como visto anteriormente em outros cursos, um site é formado por vários elementos que desempenham diferentes funções.

  • HTML é uma linguagem de marcação que confere significado ao conteudo de um site, funcionando como seu esqueleto.
  • CSS é uma linguagem de estilos que permite estilização aos conteudos de um site, dando-lhe sua aparência final.
  • JavaScript é uma linguagem de programação que confere comportamento aos conteudos de um site, permitindo-lhe executar ações.

Para realizarmos um teste de como podemos utilizar o JavaScript, usaremos o "Olá, mundo!" para inserir uma mensagem via JavaScript no nosso HTML, que pode ser vista abaixo.

Fundamentos da linguagem JavaScript

Para programarmos em JavaScript, precisamos primeiro dominar seus fundamentos.

  • Variaveis: são espaços na são espaços na memória do computador onde você pode armazenar dados. Você começa declarando uma variável com a palavra-chave var (menos recomendado) ou a palavra chave let, seguida por qualquer nome que você queira chamá-la.
    • Para declarar uma variavel, use a estrutura let minhavariavel; (sendo o ponto-e-virgula simbolo de finalização das instruções em JavaScript).
    • Depois de declarar uma variavel, pode-se atribuir a ela um valor, pela estrutura minhavariavel = 'valor'; .
    • Ou pode fazer a declaração de sua variavel e do seu valor numa unica linha, pela estrutura let minhavariavel = 'valor'; .
    • O valor dado a minhavariavel pode mudar, a não ser que ela seja uma constante.
      • Caso a variavel mude, declare usando let.
      • Caso a variavel não mude, declare usando const.
    • Os valores dados a uma variavel podem ser:
      • string: uma sequencia de texto, que deve ser envolvido em aspas simples ('').
      • numeros: não precisam ser envolvidos por aspas simples.
      • boolean: Um valor verdadeiro ou falso. São palavras-chaves especiais em JS e não precisam de aspas.
      • array: uma estrutura que permite armazenar vários valores em uma única variável ao envolver este valores entre colchetes separados por virgulas [ , , ,].
      • object: basicamente, qualquer coisa. Tudo em JavaScript é um objeto e pode ser armazenado em uma variável.
    • Serve para fazer qualquer coisa interessante em programação ao criar dinamicidade no seu projeto.

  • Comentários: Os comentários são, essencialmente, pequenos trechos de texto que podem ser adicionados entre os códigos e são ignorados pelo navegador. Podem ser adicionado pela estrutura /* mensagem */ ou // mensagem (caso não tenha quebra de linha.)

  • Operador: é um símbolo matemático que produz um resultado baseado em dois valores (ou variáveis). Alguns deles são:
    • Adição ( + ), Subtração ( - ), Multiplicação ( * ) e Divisão (/), que operam exatamente como na matematica.
    • Atribuição ( = ), que associa um valor a uma variável.
    • Operador de igualdade ( === ), que testa dois valores para ver se eles são iguais, retornando um resultado true/false (booleano).
    • Negação ( ! ) ou não igual ( !== ), que retorna o valor lógico oposto do sinal, transformando um true em um false, etc. Quando usado junto com o operador de igualdade, o operador de negação testa se os valores são diferentes.

  • Condicionais: são estruturas de código que te permitem testar se uma expressão retorna verdadeiro ou não, executando um código alternativo revelado por seu resultado.
    • Vai se expressar por uma operação que terá como resultado um valor boleano em relação a uma suposição. Veja os exemplos abaixo:
      • Uma const é par. Para sabermos isso, precisamos criar uma expressão que defina um valor como par, que é se divisivel por 2.
        Podemos expressar isso através das estruturas "const numero = x;" e "const par = numero % 2 === 0;", no qual a constante numero é o valor que desejamos saber se é par e a constante par é o que define um numero como par.
        Se o resultado da operação for 0, ele é verdadeiro. Se não for, ele é falso.
      • Então a condicional testa as variaveis sobre alguma suposição dentro de uma operação, o que significa que precisa de variaveis e uma definição expressa em operação para funcionar.
    • Na condicional if (...) {comando}, a expressão dentro do parenteses é o teste. Se esse teste retorna true, o primeiro bloco de código será executado. Se não for verdadeira, o primeiro bloco de código será ignorado e o segundo bloco de código, após a instrução else, será executado.
    • A negação da condicional if (...) {comando} é if (!...) {outro comando}, no qual ! funciona como operador de negação.
    • Para abrevear a transcrição da negação completa, pode-se incluir a palavra else após as chaves, economizando em linhas de codigo. Assim a expressão de negação se torna if (...) {comando} else {outro comando}.
    • Para escalar as condicionais, pode-se usar a expressão if (...) {comando} else if (...), {outro comando} else {mais outro comando}.
    • Por convenção, se coloca o prefixo is nas denominações de variaveis boleana.
    • Uma variavel criada dentro de um bloco de condicional só será válida dentro daquele bloco. Preferir este formato caso só necessite uma vez da variavel.

  • Funções: são uma maneira de encapsular funcionalidades que você deseja reutilizar. Quando você precisa de um procedimento, você pode chamar a função com um nome, em vez de reescrever o código inteiro a cada vez.
    • Se você ver algo que parece com um nome de variável, mas com parênteses — () — depois, provavelmente é uma função.
    • Funções geralmente tem Argumentos — pedaços de dados que elas precisam para realizarem o seu trabalho. Os argumentos são colocados dentro dos parênteses e separados por vírgulas, se houver mais de um. Basta construir o argumento através da seguinte estrutura:
      • function nomedafuncao (valor1, valor2) {
        let resultado = valor1 op valor2;
        returne resultado;
        }

        function nomedafuncao (valor1, valor2) {
        let resultado = valor1 op valor2;
        console.log(resultado);
        }
      • Use return para calcular um resultado que não será apresentado no final.
      • Use a função console.log() para apresentar o resultado final.

  • Eventos: são estruturas de código que percebem as coisas que acontecem no navegador, executando o código em resposta.

Simplificando o codigo

Para tornar o codigo mais simples e mais facil de ser lido, é possivel escrever estruturas que servirão de repositorio de dados que poderam ser invocados adiante no codigo. Elas são:

  • Objeto: é uma coleção dinamica de dados formada por chaves ({}) e valores.
    • Possui a seguinte estrutura:

      const/let nomeobjeto = {
      valor1 = x
      valor2 = y
      }

    • Pode se incrementada, adicionando um valor ao objeto por ponto final: nomeobjeto.y = z;
    • Pode deletar valor tbm, usando delete nomedoobjeto.valor1.
    • É possivel referenciar um valor especifico através da sintaxe ${this.valor};
    • Pode-se referenciar um valor especifico por nomedoobjeto['valor1'] = valor1.

  • Classes é uma estrutura de cogigo que define genericamente algo.
    • Aglutina valores que descrevem o que algo é de modo a classifica-lo. É uma definição ampla de um conceito.
    • Difere de uma instancia, que é uma definição especifica de um objeto que pertence a uma classe, uma ocorrencia da classe.
    • Exemplo:
      Pessoa {nome; idade; peso; altura, profissao}; const tamires{nome = 'Tamires'; idade = 29; peso = 49; altura = 1.59; profissão = 'designer'}; (Instancia de Pessoa)
    • Para simplificar, podemos invocar a função constructor, que criará uma instancia a partir da correspondencia de seus valores com os atributos que constroem uma classe. Assim, variaveis podem ser contextualizadas.

    • Use a função new para especificar um objeto como classe.
      Exemplo: const tamires = new Pessoa ('Tamires', 29, 49, 1.59, designer)
    • Metodo é uma instrução dentro do objeto (uma função interna).
    • Verificar exercicio do Desafio 4 para melhor compreensão dos exemplos.

  • Arrays (ou listas) são blocos de codigo que sinalizam um conjunto de valores, que podem ser referenciados dependendo de suas posições. Segue a estrutura:

    variavel = [valor1, valor2, valor3];

    • Usa-se colchetes ([ ]) para sinalizar um array;
    • A indicação da posição de um valor em uma lista começa do zero (0), como variavel[0].
    • Pode-se adicionar valores ao array usando a estrutura variavel.push(valor4); ou variavel[3] = valor4;
    • Adição por push inclui o valor ao final da lista. Caso adicione um valor diretamente pela posição, o valor colocado por push será substituido e apagado.
    • varial.pop(); remove o ultimo valor de uma lista.
    • variavel.shift(); remove o primeiro.
    • variavel.lenght(); mostrar o tamanho da lista, quantos valores ela possui.

  • Estruturas de repetição servem para repetir instruções ao codigo.
    • for é uma estrutura que serve para percorrer uma instrução um numero determinado de vezes. Segue a estrutura:

      for (let i = 0; i < array.length; i++) {}

      na qual let i é a declaração da veriavel com a posição desejada, i < array.leght é o tamanho da lista e i++ é o incremento a declaração, que no caso é +1.

Importando e exportando com JavaScript

É possivel compartilhar estruturas de codigo entre diferentes arquivos .js. Para isso, basta implementar estruturas dentros destes arquivos que permitam que eles importem e exportem estes codigos.

  • Exportar: crie as estruturas que deseja exportar e aglutine como valores dentro do objeto module.export { estr1, estr2 }
  • Importar: crie no novo arquvivo uma variavel que utilize como valor require('enderecodoarquivoexportado');
