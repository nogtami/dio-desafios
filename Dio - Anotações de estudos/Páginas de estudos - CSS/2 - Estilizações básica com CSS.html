Estudos de CSS - Estilizações básicas

Neste documento, apresento as anotações dos aprendizados adquiridos durante o curso de Estilizações Básica com CSS, da Dio.me.

Cores

Existem várias possibilidades de configuração de cores no CSS. Algumas delas são:

  • Cores pré definidas:são cores oferecidas pelos navegadores de forma pré-definida, as quais os desenvolvedores conseguem referencia-las pelo nome.
    Exemplo: aquamarine, lightblue, lightgreen.
    • Palavra reservada "currentcolor: aplica a configuração atual da propriedade color as outras propriedade de um elemento
    • Palavra reservada "transparent": Deixa qualquer propriedade de cor escolhida transparente.

  • Hexadecimal: um sistema que combina seis numeros para criar cores, variando entre numeros que vão de 0 a 9 e letras que vão de A a F.
    • Para trabalhar com a opacidade, basta adiocionar valores entre 00 e FF ao final dos seis numeros.

  • RGB:utiliza o sistema RGB (Red, Green, Blue) para criar combinações de cores.
    • Utiliza valores numero do 0 ao 255 para definir as tonalidades de cada cor primária.
    • Também defni valores através de porcentagens.

  • RGBA: é uma extensão do sistema RGB, com a adição de um canal Alfa que definirá a opacidade das cores.
    • O valor de opacidade é declarado por decimais que variam de 0.0 a 1.0.

  • HSL: sistema de combinação de cores que trabalha com Matriz (Hue), Saturação (Saturation) e Brilho (Lightness).
    • Matriz define o grau na cor no circulo cromatico (valor em grau), estabelecendo seu tom.
    • Saturação define a escala de cinza da cor (valor em %), estabelecendo sua vivacidade.
    • Brilho define a quantidade de branco de uma cor (valor em %), deixando-a mais clara ou escura.

  • HSLA: é uma extensão do sistema HSL, com a adição do canal Alfa que estabelece a opacidade da cor.
    • O valor de opacidade é declarado por decimais que variam de 0.0 a 1.0.

Imagens

Existem duas propriedades no CSS que são fundamentais para trabalhar com imagens no documento HTML.

  • Object-fit: determina o redimensionamento de uma imagem dentro de um elemento, definido como container.
    Essa propriedade vai dizer para a imagem ou video de ele deve preservar a sua proporção ou se distorcer para caber no container.
    • Fill: redimensiona a imagem para que ela caiba no container, seguindo os valores de proporção estabelecido ao container.
    • Contain: redimensiona a imagem proporcionalmente para que ela caiba no container.
    • Cover: Corta as laterais da imagem para que ela caiba no container.
    • None: a imagem ignora as proporções estabelecidas ao container e mantem suas proporções originais.
    • Scale-down: irá escolhe das configurações anteriores aquela que apresenta os menores valores adequados ao container.

  • Object-position: determina como um objeto deve ser posicionado horizontal e verticalmente dentro de um container.
    Trabalha em conjunto com a propriedade object-fit.
    • Utiliza valores em porcentagem (%) ou palavras reservadas (top, left, center, right, bottom).

Fundo dos Elementos

É possível alterar o fundo do elementos HTML acrescentando cores, imagens, gradientes, etc.

  • background: funciona como uma propriedade resumida que serve como atalho para configurar valores de outras propriedades de fundo no CSS.
    • Basta adicionar os valores separados por espaço ( ).

  • background-color: define a cor de fundo de um elemento.

  • background-image: define uma imagem pre-definida (url) ou constroi um gradiente como plano de fundo de um elemento. (linear/radial-gradient).

  • background-size: ajusta imagens ao elemento que estará contida.
    • Auto: ajustará a imagem automaticamente ao elemento. (Padrão)
    • Cover: cobrirá o elemento com a imagem, se ela for maior será cortada e se for menor, será expandida.
    • Contain: redimensionará a imagem para que ela apareça de forma completa no elemento.
    • Valores (px,%): ajusta as imagem seguindo valores para largura e altura do elemento.
    • Por camadas: separando valores por virgula (,), ajusta as dimensões de diferentes imagens num elemento.

  • background-repeat: define se haverá repetição da imagem em um elemento.
    • Repeat: repetirá a imagem lado a lado em todas as direções. (Padrão)
    • Repeat-x: define que a imagem só deve se repetir no eixo x (largura).
    • Repeat-y: define que a imagem só deve se repetir no eixo y (altura).
    • Space: repetira a imagem de forma a manter sua integridade, adicionando espaço vazios ao elemento para garantir isso.
    • Round: redimensiona a imagem para que ela caiba integralmente no elemento, sem a necessidade de corte.
    • No-repeat: define que a imagem não deve ser repetida no elemento.
    • Todos este valores podem ser aplicados por eixo (largura x altura).

  • background-position: define o posicionamento da imagem dentro do elemento.
    • Pode utilizar valores em pixels (px), porcentagens (%) ou nomenclaturas (top, left, right, bottom, center).
    • Estes valores podem ser declarados por eixo.

  • background-attachment: define como o fundo do elemento vai se mover em relação a rolagem do navegador.
    • fixed: define a imagem de fundo como fixa mesmo que rolemos os conteudos do elemento e a pagina do navegador.
    • scroll: define que a imagem de fundo role junto com o navegador, mas se mantenha fixo em relação aos conteudos do elemento.
    • local: define que os conteudos do elementos rolem junto com a imagem de fundo, mas se mantenha fixos com a rolagem do navegador.

  • background-origin: define qual será o ponto de origem de uma imagem de fundo.
    • padding-box: define que o ponto de origem da imagem de fundo seja no canto superior esquerdo do seu espaçamento interno. (Padrão)
    • border-box: define que o ponto de origem da imagem de fundo seja o canto superior esquerdo da borda do elemento.
    • content-box: define que o ponto de origem da imagem de fundo seja o canto superior esquerdo da área de conteudo.

  • background-clip: permite definir o prenchimento de fundo para cores e imagens num elemento.
    • padding-box: permite preencher a área de espaçamento interno de um elemento.
    • border-box: permite preencher a área de borda de um elemento.
    • content-box: permite preencher a área de conteudo de um elemento.
    • text: permite preencher as cores do texto de conteudo com o fundo do elemento. (No Chrome, é necessario configurar -webkit-background-clip: text para funcionar).

  • background-blend-mode: permite realizar mesclagens entre cor e imagem de fundo num elemento.
    • São muitas possibilidades de valores para esta propriedade, por isso é válida uma pesquisa só sobre isso.
    • Funciona para adicionar efeitos nas imagens.

Bordas

Pode-se adicionar e configurar bordas nos elementos de um documento HTML utilizando algumas propriedades CSS.

  • border: é uma propriedade resumida que serve como atalho para configurar valores de outras propriedades de bordas CSS.
    • Basta adicionar os valores separados por espaço ( ).

  • border-width: define a largura da borda de um elemento.
    • Seus valores podem definidos por pixels (px), porcentagens (%) ou nomenclaturas (thin, medium, thick)
    • Pode definir valores por eixo (largura x altura, cima/direita/baixo/esquerda) separando por espaço ( ).

  • border-style: define o tipo de borda que colocaremos no elemento.
    • none: define que não há bordas nos elementos (Padrão);
    • solid: define uma borda por linha contínua;
    • dashed: define uma borda por traços;
    • dotted: define uma borda pontilhada;
    • double: define uma borda dupla;
    • groove: define uma borda superior tridimensional;
    • ridge: define uma borda inferior tridimenshional;
    • inset: define uma borda de aprofundamento do elemento;
    • outset: define uma borda de destaque do elemento.
    • Pode definir valores por eixo (largura x altura, cima/direita/baixo/esquerda) separando por espaço ( ).

  • border-color: define a cor da borda de um elemento.
    • Segue como unidades de valores todas as unidades de cores discutidas anteriormente.
    • Caso não tenha definido uma propriedade especifica de cor para borda, seguirá a propriedade color. (Padrão)
    • Pode definir valores por eixo (largura x altura, cima/direita/baixo/esquerda) separando por espaço ( ).

  • border-radius: define o raio da borda, arredondando seus cantos.
    • Pode ser aplicado sem haver uma borda visivel, dando a impressão que o próprio elemento é arredondado.
    • Pode definir valores por eixo (largura x altura, cima/direita/baixo/esquerda) separando por espaço ( ).
    • Para um efeito oval, definir valores divididos por /.
    • Define valores por pixels (px) ou porcentagem (%).

  • border-image: é propriedade resumida que serve para configurar valores de outras propriedades que define bordas para imagens no CSS.
    • Basta adicionar valores separados por espaço ( ).
    • Para separar os valores referentes ao width, outset e slice, utiliza barra (/).

  • border-image-source: define o caminho de imagem que será utilizada para criar uma borda ao elemento.
    • Aceita o valor none. (Padrão)
    • Define uma borda usando o caminho de uma imagem (valor url) ou criando um gradiente (linear/radius-gradient).
    • Sem outras propriedades, irá preencher com a imagem ou o gradiente somente os cantos da borda do elemento.

  • border-image-width: define qual será a largura de imagem da borda de um elemento.
    • Pode definir valores por eixo (largura x altura, cima/direita/baixo/esquerda) separando por espaço ( ).
    • Seus valores podem definidos por pixels (px), porcentagens (%) ou nomenclaturas (auto)

  • border-image-repeat: define a possibilide de repetição da imagem na composição da borda de um elemento.
    • repeat: divide a imagem para preencher toda a área das laterais da borda.
    • strech: estica as regioes laterais da imagem para que ocupe todo espaço da borda. Não mexe nos cantos.
    • round: vai redimensionar a imagem de modo a evitar cortes.
    • space: redimensiona a imagem para que ela caiba na borda, criando espaços caso a imagem não possa se ajustar por completo as bordas.
    • Pode definir valores por eixo (largura x altura, cima/direita/baixo/esquerda) separando por espaço ( ).

  • border-image-outset: definira a distância entre borda e elemento.
    • Seus valores podem definidos por pixels (px) ou porcentagens (%), sempre positivos.
    • Pode definir valores por eixo (largura x altura, cima/direita/baixo/esquerda) separando por espaço ( ).

  • border-image-slice: divide a imagem em regiões para serem trabalhadas na borda de um elemento.
    • Os cantos da imagem são as regiões 1, 2, 3 e 4 (em sentido horário) e são utilizadas uma vez para compor a imagem final da borda. Ficaram nos cantos do elemento.
    • As regiões 5, 6, 7, 8 foram as laterais horizontais e verticais da imagem (direita, esquerda, cima, baixo) e podem ser repetidas ou modificadas para formar a borda.
    • A região 9 representada a região central da imagem, que é descartada para formar a borda do elemnto.
    • As três propriedades de bordas acima definem como estas regiões serão utilizadas para definir a borda do elemento.
    • Pode-se utilizar valores pixels (px), porcetagem (%) ou fill (um valor que preenche o centro do elemento com a região que anteriormente seria descartada).
    • Aceita mais de um valor, separando-os por espaço ( ).

Fontes

Pode-se modificar a tipografia dos elementos textuais de um documento HTML utilizando-se algumas propriedades CSS.

  • font é uma propriedade resumida que serve como atalho para configurar valores de outras propriedades de fonte CSS.
    • Basta adicionar valores separados por espaço ( ).
    • Os valores do font-size e font-family são obrigatórios e, caso não seja definidos pelo desenvolvedor, serão definidos pelo navegador.
    • Os valores de font-size e line-height devem ser separados por barra (/).
  • font-family: define uma fonte especifica ou uma familia de fonte (padrões do sistema operacional) para os elementos tipograficos do documento HTML.
    • Como boas práticas, fontes que tenham como nome mais de uma palavra separada por espaço devem ser apontadas entre aspas ("").
    • Caso o navegador não possua a primeira opção de tipografia, pode-se definir uma segunda, separando por vírgula (,). (técnica fallback)

  • font-weight: define a espessura das fontes de um elemento.
    • Pode-se definir por nomenclatura (bold, italic, thin,...)
    • A tag b nada mais é do que a aplicação do font-weight bold dentro do próprio arquivo HTML.
    • Aceita valores de 100 a 900 e nomenclaturas (normal, bold, lighter, bolder).

  • font-size: define o tamanho de uma fonte.
    • Aplica tanto valores númericos (em px ou outras unidades de medida) quanto nomenclatura ( xx-small, x-small, small, medium, large, x-large, xx-large, smaller, larger).

  • font-style: define o estilo de uma fonte (normal, itálica ou obliqua)
    • normal: estabelece a fonte sem inclinações. (Padrão)
    • italic: estabelece a fonte com inclinações sutis.
    • obliqua: estabelece a fonte com inclinações evidentes.
    • A tag i nada mais é do que a aplicação do font-style italic dentro do próprio arquivo HTML.

  • font-variant: define se o texto será exibido num formato versallete (small caps).
    • Aceita dois valores: normal ou small-caps.

  • font-stretch: define se o texto será mais condesado ou expandido.
    • Só funciona se a fonte selecionada possuir esta configuração.
    • Não é muito utilizada porque muitas fontes não dão suporte a ela.
    • Seus valores podem ser aplicados por nomenclatura (ultra-condensed, extra-condensed, semi-condensed, normal, semi-expanded, expanded, extra-expanded, ultra-expanded).

  • line-height: define os espaçamentos entre linhas num texto, tomando como referencia o tamanho da fonte.
    • Aceita valores em pixels (px), porcentagens (%), nomenclatura (normal) ou números sem unidades de medida.

  • @font-face: define uma fonte que não está instalada no sistema operacional do desenvolvedor ou do usuário.
    • Estabelece uma familia de fonte e seu caminho (url).
    • Pode-se utilizar o caminho local ( ) para indicar que a fonte customizada deve ser buscada no sistema operacional também.
    • Pode estabelecer configurações diferentes e independentes do sistema operacional para as tipografias. Serve para que o navegador configura a fonte de modo originalmente idealizado pelo seu criador, sem adaptações do navegador.
    • Pode definir de forma independente o font-weight, font-strech e font-style.

  • @input url(): importa uma folha de estilo para o arquivo CSS, que podem estar presentes tanto no nosso arquivo quanto num servidor remoto.
    • Caso haja alguma alteração no arquivo CSS que estamos chamado, ela se refletirá no arquivo CSS importado para o projeto.
    • Outro metodo de importação de fonte é através de link, que deverá ser colado ao head do documento HTML. Pode trazer dificuldades a manutenção de projetos com muitas páginas.
    • Dica: criar uma arquivo CSS global e importa-lo com todos os estilos de fontes que serão utilizados no projeto para facilitar a manutenção da tipografia.

Texto

Existem algumas propriedades no CSS que são responsáveis por estilizar textos. Algumas delas são:

  • text-transform: define quais caracteres estarão maiusculo, minisculos, etc.
    • none: não aplica nnenhuma modificação ao texto. Mantem o texto do modo que foi digitado. (Padrão).
    • capitalize: define o primeiro caractere de um texto em maiusculo.
    • uppercase: define todos os caracteres do texto como maiusculos.
    • lowercase: define todos os caracteres do texto como minusculos.
    • initicial: aplica o valor padrão automaticamente.
    • inherit: define o valor do elemento com base no elemento-pai.

  • text-align: define o alinhamento do texto dentro do elemento.
    • left: alinhamento à esquerda. (Padrão)
    • right: alinamento à direita.
    • center: alinahemento ao centro.
    • justify: alinhamento justificado.

  • text-decoration: é uma propriedade resumida que serve para configurar outras propriedades de adição de linhas ao texto.
    • Basta adicionar valores separados por espaço ( ).

  • text-decoration-line: define a posição de linha no texto.
    • none: define o texto sem linhas. (Padrão)
    • underline: define uma linha para sublinhar um texto ou palavra.
    • line-thorught: define uma linha riscando o texto.
    • overline: define uma linha acima do texto.

  • text-decoration-style: define o tipo de linha que será colocado no texto.
    • solid: adiciona uma linha contínua ao texto.
    • double: adiciona uma linha dupla.
    • dotted: adiciona uma linha pontilhada ao texto.
    • dashed: adiciona uma linha traçada.
    • wavy: adiciona uma linha ondulada.

  • text-decoration-color: define a cor da linha adicionada ao texto.
    • Segue os sistemas de cores visto anteriormente.

  • text-decoration-thickness: define a espessura da linha adicionada ao texto.
    • Aceita valores em pixels (px).

  • text-ident: define o recuo da primeira linha do texto.
    • Aceita valores númericos, positivos ou negativos.

  • letter-spacing: adiciona espaçamento entre os caracteres de um texto.
    • normal: não adiciona espaço entre letras. (Padrão)
    • Aceita valores numericos, positivos e negativos.

  • word-spacing: adiciona espaçamento entre cada palavra de um texto.
    • normal: não adiciona espaço entre letras. (Padrão)
    • Aceita valores numericos, positivos e negativos.

  • white-spacing: define como os espaços em branco de um texto serão tratados.
    • normal: combina sequência de espaços no documento HTML em apenas um espaço e as quebras de texto são realizada no limite do elemento. (Padrão)
    • nowrap: combina sequencia de espaços no documento HTML em apenas um espaço e não realiza quebra de texto.
    • pre: preserva sequencia de espaços em branco e as quebras de texto só ocorrem quando há o fim de frases.
    • pre-line: combina sequencia de espaços em branco e realiza quebras de texto ao final de frase e nos limites do elemento.
    • pre-wrap: preserva sequencia de espaços em branco e realiza quebras de texto ao final da frase e nos limites do elemento.
    • break-spaces: faz o mesmo que o pre-wrap, mas quebrao texto quando a sequencia de espaços em branco ultrapassa os limites do elemento.

  • word-wrap: define se uma palavra deve ser quebrada entre duas linhas ou se ela deve ser mantida integralmente numa linha só.
    • normal: não realiza quebra de palavra. (Padrão)
    • break-word: quebra a palavra em qualquer caracter para que ela caiba no elemento. Preserva quebra de linhas.

  • word-break: define quando no texto deve ter uma quebra de linha.
    • normal: as quebras de linhas no texto ocorrem quando há um espaço em branco ou hifen. (Padrāo)
    • break-all: define que o texto será quebrado em qualquer parte. Não preserva quebra de linhas.
    • keep-all: define que o texto não tenha quebra de linhas.

  • writing-mode: define a orientação do texto (vertical ou oriental).
    • horizontal-tb: define que o texto siga a orientação de leitura horizontal, da esquerda para a direita. (Padrão)
    • vertical-rl: define que o texto siga verticalmente, da direita para esquerda.
    • vertical-lr: define que o texto siga verticalmente, da esquerda para direita.

  • text-overflow: define como será mostrado ao usuário a existencia de mais conteudo do que ele consegue ver em um container.
    • clip: o texto é cortado e nenhum outro elemento é adicionado para indicar que há mais conteudo. (Padrão)
    • ellipsis: o texto é cortado e é adicionado reticencias para indicar que há mais conteudo num container.
    • string: adiciona um texto personalizado entre aspas ("") para indicar que há mais conteudo num container. Só funciona no Firefox.

Sombras

No CSS, existem algumas propriedades que permitem que adicionemos sombras em elementos ou textos.

  • box-shadow: adiciona sombras em elementos gráficos.
    • Aceita valores para propriedades distintas de sombras, como posição, cor, desfoque e propagação. Basta separar por espaço ( ).
    • A sequencia de definição de valores numerico seguirá a seguinte ordem: horizontal, vertical, desfoque, propagação.
    • Aceita valores numericos positivos e negativos.
    • Outset define uma sombra externa (Padrão) e inset define uma sombra interna.
    • Aceita varias camadas de sombra, basta separa-las por virgula (,).
    • Com a propriedade filter=drop-shadow ( ) é possivel criar sombras dentro de uma imagem, não tratando como um container.

  • Text-shadow: aplica sombras em textos, caracter por caracter.
    • Aceita valores para propriedades distintas de sombras, como posição, cor e desfoque. Basta separar por espaço ( ).
    • A sequencia de definição de valores numerico seguirá a seguinte ordem: horizontal, vertical e desfoque.
    • Aceita valores numericos positivos e negativos.
    • Aceita varias camadas de sombra, basta separa-las por virgula (,).

Outras propriedades

  • Opacity: define a opacidade dos elementos.
    • Aceita valores numericos decimais de 0.0 a 1.0.

  • Overflow: controla como o conteudo deve ser exibido caso ele ultrapasse os limites de seu container.
    • visible: o conteudo não é cortado e continua sendo exibido mesmo que ultrapasse o container. (Padrão)
    • hidden: o conteudo será cortado e o que estiver fora do container não será exibido.
    • scroll: o conteudo será cortado, mas poderá ser acessado através de um scroll.
    • auto: coloca uma barra de rolagem automaticamente caso o conteudo ultrapasse o container.
    • É necessária a definição de uma altura para que funcione, pois, do contrário, o container crescerá para encapsular o conteudo.
    • Pode definir um scroll vertical ou horizontal independentes, atraves do overflow-x e overflow-y.

Redefinindo propriedades de navegadores

Reset CSS é uma folha de estilos que irá sobrepor todas as propriedades de estilo CSS que os navegadores colocam por padrão nos elementos, ajudando que nossos projetos tenham o mesmo design em todos os navegadores.

  • A folha de estilo Reset CSS mais famosa e facilmente encontrada na internet eé de Erik Mayer.
  • Pode-se usar o seletor universal para definir as propriedades que serão resetadas em todos os elementos.
  • Também pode ser usada importando o arquivo reset.css para dentro de um outro arquivo css.

Outra forma de redefinir propriedades dos navegadores é atraves de um normalize, um documento CSS que padroniza todo estilo inicial de um site.
Disponiveis em Normalize CDNjs. Utilize o link disponivel no head da página.

Responsividade

Para ajustar seu layout de acordo com os tamanhos de tela, use @media screen. Este atalho permite adequar a forma como o conteudo se apresentará na tela dependendo do seu tamanho. Os principais tamanhos são:

  • xs (for phones - screens less than 768px wide);
  • sm (for tablets - screens equal to or greater than 768px wide);
  • md (for small laptops - screens equal to or greater than 992px wide);
  • lg (for laptops and desktops - screens equal to or greater than 1200px wide).
