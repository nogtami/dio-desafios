O que é CSS?

Cascating Style Sheets (ou Folha de Estilo em Cascata) é um mecanismo para adicionar estilos para um documento Web (html).

  • Criado por Hakon Wium Lie em 1994 para facilicar na programação de sites. Atualmente está na sua versão CSS3.
  • Ele determina como vai ser o layout de uma pagina e como os seus elementos devem ser.
  • Permite controlar o estilos de várias páginas html com um só documento CSS.
  • Permite criar layout simples com poucos códigos.
  • Apresenta um conceito flexivel que permite criar não somente aplicações web, mas também para aplicativos móveis através de bibliotecas/frameworks.

Não é uma linguagem de programação e nem de marcação. O CSS é uma linguagem de estilos

Combinado com o HTML, permite a criação de:

  • Layout e estilizações de página web;
  • Animações;
  • Formas geometricas e desenhos;
  • Filtros;
  • Contadores.

O que são propriedades e valores?

Propriedades são características de um elemento do nosso HTML (cor de fundo, largura, altura, espaçamento...). Valores definem o resultado de uma propriedade e como o navegador deve exibir o estilo daquele elemento. Deve ser exibido da seguinte forma: Propriedade: valor, como color: red ou background: green.

Formas de declarar o CSS

  • CSS Inline: adiciona o código CSS utilizando o atributo style dentro das tags HTML, elemento por elemento.
    • Não é recomendada por dificultar padronização de elementos e exigir muito esforço em caso de modificação (necessidade de corrigir elementos um por um).
    • Pode ser vantagoso para modificar estilos em documentos HTML que você não possua acesso ou que você precisa subescrever estilos aplicando em algum outro documento CSS.
    • Pode ser vantajoso para testar e visualizar alterações de estilo de forma rápida na nossa página.
    • Diminui solicitações HTTP porque não precisa fazer download de um arquivo CSS externo. Já está embutido no HTML.

  • CSS Interno: adiciona o código CSS dentro da tag head do HTML, a qual adicionamos a tag style e colocamos as regras CSS nesta área.
    • Facilita a velocidade de carregamento do site porque o CSS está embutido no mesmo arquivo HTML.
    • Não é possivel reaproveitar este código de estilos em outras páginas HTML, o que dificulta a manutenção da página e a utilização inteligente dos estilos.

  • CSS Externo: é criado um arquivo com a extensão .css com todas as regras do CSS que queremos implementar. Este arquivo é referenciado no HTML da página através da tag link adicionada ao head do arquivo HTML.
    • Como boa prática, mantém o arquivo CSS numa página chamada "assets".
    • Facilita a padronização de estilos de multiplas páginas.
    • Facilita o processo de manutenção do site e torna o processo de desenvolvimento do seu código mais produtivo.
    • Mantém as paginas HTML menores e com estruturas mais limpas e organizadas.
    • Velocidade de carregamento de todas as páginas HTML é mais rápida.
    • Como desvantagem, pode haver um delay no carregamento de estilos das paginas HTML referenciadas, pois necessita do download de estilos de um arquivo externo. Em outras palavras, pode ser que haja uma diferença pequena no carregamento dos seus arquivos HTML e CSS, o que pode deixar sua página sem estilos por alguns instantes. Depende da conexão do usuário e pode ser minimizado.

Depurando o CSS

O processo de depuração (ou debug) é uma forma de identificar problemas no código-fonte de uma aplicação e entender seu comportamento.
Todos os navegadores possuem uma ferramenta chamada Dev Tools que permite que desenvolvedores possam examinar nossos sites, requisições para recursos externos, etc.
Esta ferramenta pode (e deve!) se utilizada para o desenvolvimento das nossas aplicações web para que possamos visualizar as configurações do nosso códigos e como ele se comporta em diferentes dispositivos. Também é possível realizar edições temporárias que ajuda na visualização de algumas modificações e como os elementos se adequam a elas.

Seletores

No CSS, há os seletores, que são indicadores que definem em quais elementos serão aplicadas as propriedades e valores estabelecidos para estilizar o arquivos HTML.
São eles:

  • Seletor de Tag (ou Tipos), que busca elementos por uma tag HTML especifica;

  • Seletor de ID (#), que busca os elementos pelo atributo id;
    • Não pode repetir o mesmo nome para diferentes id. É como uma impressão digital dentro do nosso código.

  • Seletor de Class (.), que busca elementos através do atributo class;
    • Class é um atributo que classifica grupos de elementos. Então, tags diferentes podem ter atributos class com o mesmo nome.
    • Pode selecionar uma ou mais classes.

  • Seletor Universal (*), que seleciona todos os elementos HTML;

  • Seletor de Atributo ([atrib]), que seleciona elementos que possuem um atributo especifico no nosso HTML e também busca atributos com um valor especifico.
    • Modifica todos os atributos identicos quando colocado como [atrib];
    • Modifica todos os atributos de mesmo valor quando colocado [atrib=value];
    • Modifica atributos por palavra-chave quando colocado [atrib~=value];
    • Modifica atributos por palavras isoladas ou com hifen quando colocado [atrib|=value];
    • Modifica atributos por prefixos quando colocado [atrib^=value];
    • Modifica atributos por sufixos quando colocado [atrib$=value];
    • Modifica atributos com valor localizado em qualquer parte quando colocado [atrib*=value].

Combinadores

Podemos aglutinar regras de CSS de algumas formas.

  • Agrupadores de seletores: Podemos aplicar as mesmas regras CSS para vários seletores diferentes. Para isso, podemos separar os seletores utilizando uma vírgula (,). Assim não é necessário ficar copiando e colando os mesmos códigos para todos. Basta fazer um só bloco de CSS utilizando as mesmas regras.
    • A virgula funciona semanticamente como um "ou".
    • Para juntar seletores criando uma condição de adição, aglutinamos seletores sem espaços e vírgulas. br Semanticamente, estamos definindo que a regra CSS vai funcionar se aquele elemento for/tiver uma condição e outra.

  • Combinador Descendente: Busca por todos os elementos inseridos numa tag maior. Separa-se por espaço ( ).
    Exemplo: li li para alterar estilos dentro de subitens.

  • Combinador Filho: Busca por elementos desejados que são filhos diretos do elemento que queremos modificar. Separa-se por maior que (>).
    Exemplo: div > p vai modificar só os paragrafos que estão relacionados diretamente com div.

  • Combinador Irmão adjacente: Busca por elemento que está diretamente ao lado de outro. Separa-se por mais (+).
    Exemplo: div + p vai modificar só paragrafos que estão diretamento ao lado de div.

  • Combinador Irmão em geral: Busca por todos os elementos irmão a outros, independente de estarem aos lado deste. Separa-se por til (~).
    Exemplo: div ~ p vai modificar todos os paragrafos que são irmão de div.

Propriedade de dimensionamento e espaçamento

Conhecendo um pouco sobre as principais caracteristicas e indicadores do CSS, pode-se aprofundar um pouco mais em algumas propriedades e valores que esta linguagem de estilos oferece.

  • Propriedades de dimensionamento: São propriedades responsaveis por alterar o tamanho dos elementos (largura ou altura). No CSS, há várias unidades de medidas que podem ser utilizadas para determinar os valores das propriedades de dimensionamento.
    • width: para alterar a largura;
    • height: para alterar a altura;
    • min-width para definir uma largura minima a um elemento;
    • max-width para definir uma largura máxima a um elemento;
    • min-height para definir uma altura minima a um elemento;
    • max-height para definir uma altura máxima a um elemento.

  • Propriedades de espaçamento: São propriedades responsaveis por definir e alterar os espaçamentos dos e entre os elementos HTML.
    • margin: cria um espaçamento em volta dos elementos. (Espaçamento externo)
      • Pode ser trabalhado de forma geral (margin) ou por cada uma de suas laterais (top, left, right ou bottom);
      • Pode ser aplicados valores positivos e negativos;
      • Correspondencia de definições com um: todos os quatro lados terão o mesmo tamanho;
      • Correspondencia de definições com dois valores: cima/baixo e laterais;
      • Correspondencia de definições com três valores: cima, lados e embaixo;
      • Correspondencia de definições com quatro valores: cima, direita, embaixo e esquerda.

    • padding: cria um espaçamento entre os conteudos e bordas de um elemento. (Espaçamento interno)
      • Pode ser trabalhado de forma geral (padding) ou por cada uma de suas laterais (top, left, right ou bottom);
      • Pode ser aplicados valores positivos e negativos;
      • Correspondencia de definições com um: todos os quatro lados terão o mesmo tamanho;
      • Correspondencia de definições com dois valores: cima/baixo e laterais;
      • Correspondencia de definições com três valores: cima, lados e embaixo;
      • Correspondencia de definições com quatro valores: cima, direita, embaixo e esquerda.

    • box-sizing: estabelece criterios e limitações de altura e largura para os elementos. Possui dois valores:
      • content-box: comportamento padrao que adiciona espaço ao calcular largura = conteudo + borda + padding;
      • border-box: comportamento que redimensiona o elemento readequando o conteudo aos limites da borda, calculando largura = borda + padding.
